Lab # 6: Favicons!

What exactly is a favicon?



A favicon, also known as a shortcut icon, website icon, tab icon, URL icon, or bookmark icon,
is a file containing one or more small icons, associated with a particular website or web page.
Favicons are actually great implementations that web-designers have to use to make a page
more interactive. Browsers that do support favicons typically display a page's favicon in a
list of bookmarks

How are favicons created?

In March 1999, Micrsoft released Internet Explorer, which supported
favicons for the first time. At this time, the favicon was a file called
"favicon.log" placed in the root directory of a web site. It was mainly
used and seen in the browser's bookmarks and next to the URL in the
address bar if the page was bookmarked. Now you really don't have
to bookmark a site for it's favicon to actually show up. When starting
to create a favicon, one main thing must come into account. The format
for a favicon must be 16x16 pixels or 32x32 pixels, using 6-bit or 24-bit colors.
The format of the image must be one of PNG, GIF, or ICO. Creating large detail
images will just not work, going with something simple
that fits the pixel range and looks nice and fits with the sze is the only way a favicon can be successfully done.

How did I create the favicon for this site?

To actually create the favicon for this site, an 8-bit per pixel was used to create the colors. The image is exactly
17x17 large and fits very well when constructed to the actual size. For better reference, always make favicons
basic and easy to read when it is at actual size. The colors were all typed in binary code using the 8-bit
schmeme.
